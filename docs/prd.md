# QuickNav 产品需求文档

## 1. 文档信息

| 项目 | 内容 |
|---|---|
| 产品名称 | QuickNav |
| 产品类型 | macOS 常驻型快捷导航应用 |
| 当前阶段 | 个人使用 / 本地开发 / 不上架 App Store |
| 目标版本 | V0.1 原型到 V1.0 稳定版本 |
| 技术方向 | Swift + SwiftUI + AppKit |

## 2. 项目背景

日常工作中经常需要重复打开工作平台、启动本地应用、进入项目目录或执行固定脚本。传统方式通常依赖 Dock、Spotlight、浏览器书签、终端命令或应用切换器，路径较长且上下文切换明显。

QuickNav 通过一个可全局唤起的鼠标方向菜单，把常用动作放在鼠标附近。用户按住快捷键后移动鼠标到目标方向，松开快捷键即可执行动作，减少查找、点击和键盘输入成本。

## 3. 项目目标

### 3.1 产品目标

- 提供一个轻量、稳定、低打扰的 macOS 快捷导航工具。
- 通过全局快捷键和圆形方向菜单快速执行常用动作。
- 支持配置化管理快捷项，方便按个人工作流调整。
- 为后续扩展动作类型、设置页面和插件能力预留结构。

### 3.2 非目标

- 首版不做 App Store 发布。
- 首版不做复杂账号体系、云同步和多设备同步。
- 首版不做完整插件市场。
- 首版不强依赖视觉编辑器，允许通过 JSON 配置管理菜单项。
- 首版不保证所有系统级快捷键均可被覆盖，需遵守 macOS 权限和系统保留快捷键限制。

## 4. 目标用户和场景

### 4.1 目标用户

- 主要用户：开发者本人。
- 使用环境：macOS 桌面环境，长期后台运行。
- 使用特点：高频访问固定网站、应用、文件夹和脚本命令。

### 4.2 核心场景

1. 打开工作平台  
   用户按住快捷键，向某个方向移动鼠标，松开后打开指定 URL。

2. 启动本地应用  
   用户通过方向菜单快速打开 Xcode、VSCode、Terminal、Chrome 等应用。

3. 执行固定命令  
   用户通过菜单触发脚本，例如打开项目目录、启动本地服务或执行自定义 shell 命令。

4. 临时取消  
   用户按住快捷键后不想执行任何动作，把鼠标移回中心区域后松开，菜单关闭且不执行动作。

## 5. 核心交互

### 5.1 基础交互流程

1. 用户按下全局快捷键。
2. 应用记录当前鼠标位置作为菜单中心点。
3. 应用在中心点附近显示圆形菜单。
4. 用户按住快捷键并移动鼠标。
5. 应用根据鼠标相对中心点的角度和距离计算当前选中项。
6. 用户松开快捷键。
7. 如果当前有选中项，则执行对应动作。
8. 如果当前无选中项，则只关闭菜单。

### 5.2 取消规则

- 鼠标距离中心点小于取消半径时，不选中任何菜单项。
- 默认取消半径建议为 28px 到 40px。
- 松开快捷键时如果没有选中项，不执行动作。

### 5.3 选择规则

- 菜单项按 360 度平均分布。
- 首版支持 4 到 8 个菜单项，推荐默认 8 个。
- 角度从屏幕右侧方向开始计算，顺时针或逆时针需在实现中固定，避免配置歧义。
- 鼠标距离中心点超过取消半径后，根据角度命中最近菜单项。
- 鼠标超出菜单外圈不取消选择，仍按角度保持最近选中项。

### 5.4 边界规则

- 菜单靠近屏幕边缘时，窗口应尽量保持可见。
- 首版可以允许菜单部分贴边，但不能完全超出屏幕导致无法识别。
- 多显示器场景下，菜单应显示在鼠标所在屏幕。
- 如果无法确定屏幕，使用主屏幕坐标兜底。

## 6. 功能需求

### 6.1 应用常驻

- 应用启动后常驻菜单栏。
- 默认隐藏 Dock 图标，减少干扰。
- 菜单栏提供以下操作：
  - 启用 / 禁用 QuickNav
  - 重新加载配置
  - 打开配置文件
  - 设置
  - 关于
  - 退出应用

### 6.2 全局快捷键

- 支持后台响应全局快捷键。
- 默认快捷键：Control + Space。
- 按下快捷键显示菜单。
- 松开快捷键隐藏菜单并触发动作。
- 后续支持在设置页修改快捷键。
- 快捷键注册失败时，应在菜单栏或设置页显示错误状态。

### 6.3 圆形菜单

- 菜单显示在鼠标光标周围。
- 使用透明、无边框、置顶浮动窗口承载菜单。
- 菜单元素包括：
  - 中心点
  - 圆形背景
  - 菜单项图标或文字
  - 当前选中项高亮
  - 方向指示
- 菜单默认尺寸建议：
  - 菜单外半径：120px 到 160px
  - 菜单项尺寸：44px 到 56px
  - 中心取消区域：28px 到 40px

### 6.4 菜单动画

- 菜单出现：淡入 + 缩放。
- 菜单项出现：从中心向外展开。
- 菜单消失：淡出 + 缩小。
- 选中项反馈：放大、颜色高亮、阴影增强。
- 动画应轻量，不能影响方向选择响应。

### 6.5 动作执行

首版支持以下动作类型：

| 类型 | 说明 | 示例 |
|---|---|---|
| url | 打开网站 | `https://github.com` |
| app | 启动本地应用 | `/Applications/Visual Studio Code.app` |
| command | 执行 shell 命令或脚本 | `open ~/Projects` |

动作执行规则：

- 仅在快捷键松开时执行一次。
- 动作执行前菜单先关闭，避免视觉阻塞。
- 动作执行失败不应导致应用崩溃。
- 失败时首版可写入日志，后续在菜单栏或通知中提示。
- `command` 类型首版只面向个人使用，不做复杂沙箱隔离。

### 6.6 配置管理

- 使用本地 JSON 保存快捷键、菜单样式和菜单项配置。
- 首次启动时，如果用户配置不存在，复制默认配置。
- 配置加载失败时使用内置默认配置。
- 菜单栏支持重新加载配置。
- 配置错误应有可定位的日志信息。
- 后续设置页可以修改配置并写回 JSON。

推荐配置文件位置：

```text
~/Library/Application Support/QuickNav/config.json
```

默认配置示例：

```json
{
  "hotKey": {
    "key": "space",
    "modifiers": ["control"]
  },
  "menu": {
    "radius": 140,
    "deadZoneRadius": 36,
    "items": [
      {
        "id": "github",
        "title": "GitHub",
        "icon": "globe",
        "type": "url",
        "value": "https://github.com"
      },
      {
        "id": "vscode",
        "title": "VSCode",
        "icon": "app",
        "type": "app",
        "value": "/Applications/Visual Studio Code.app"
      },
      {
        "id": "projects",
        "title": "Projects",
        "icon": "folder",
        "type": "command",
        "value": "open ~/Projects"
      }
    ]
  }
}
```

### 6.7 设置页

V0.1 不要求设置页。V0.3 开始提供基础设置页：

- 查看当前快捷键。
- 修改快捷键。
- 查看配置文件路径。
- 打开配置文件。
- 重新加载配置。
- 显示权限状态。

### 6.8 日志和错误处理

- 配置读取失败、快捷键注册失败、动作执行失败都应记录日志。
- 首版日志可以输出到 Xcode Console。
- 后续可写入本地日志文件。
- 用户可见错误优先放在菜单栏状态或设置页内。

## 7. 权限需求

### 7.1 辅助功能权限

用途：

- 监听全局快捷键。
- 获取鼠标事件或鼠标位置。

要求：

- 应用启动时检查权限状态。
- 权限不足时提供提示，引导用户到系统设置开启辅助功能权限。
- 权限不足时菜单栏仍可用，但全局唤起不可用。

### 7.2 自动化权限

首版不强制需要。后续如果控制其他应用或执行 AppleScript，再按动作类型申请。

### 7.3 文件访问权限

用途：

- 读取和写入本地配置。
- 执行用户配置的脚本或命令。

首版以个人本地使用为前提，不做 App Sandbox。

## 8. 非功能需求

### 8.1 性能

- 常驻后台 CPU 占用应接近 0。
- 未唤起菜单时不应持续高频监听鼠标移动。
- 菜单唤起响应建议小于 100ms。
- 菜单动画应保持流畅，目标 60fps。

### 8.2 稳定性

- 配置错误时使用默认配置兜底。
- 动作执行失败不影响后续使用。
- 重复按键、快速按下松开、应用失焦等场景不应导致菜单残留。

### 8.3 可维护性

- UI、状态、系统事件、动作执行和配置管理分层清晰。
- 动作类型使用枚举或协议封装，便于扩展。
- 角度计算逻辑独立，便于单元测试。

### 8.4 可扩展性

后续可扩展：

- 打开文件或文件夹。
- 发送快捷键。
- 执行 macOS Shortcut。
- 打开项目目录。
- 多层菜单。
- 不同配置 Profile。
- 可视化菜单编辑器。

## 9. UI 需求

### 9.1 视觉原则

- 简洁、轻量、低干扰。
- 背景应半透明，不遮挡用户当前工作内容。
- 选中态必须清晰，保证快速判断当前会执行哪个动作。
- 动画可以有弹性，但不应拖慢执行。

### 9.2 菜单项展示

- 菜单项优先展示图标。
- 图标不可用时展示短文本。
- 文本建议不超过 8 个中文字符或 12 个英文字符。
- 选中时可显示标题或放大当前项。

### 9.3 空状态

- 配置中没有菜单项时，唤起菜单应显示中心提示或直接不显示动作项。
- 松开快捷键不执行任何动作。

## 10. 技术方案

### 10.1 技术栈

| 模块 | 技术 | 说明 |
|---|---|---|
| 开发语言 | Swift | macOS 原生开发语言 |
| 界面开发 | SwiftUI | 菜单、动画、设置页 |
| 系统能力 | AppKit | 窗口、菜单栏、全局事件 |
| 全局快捷键 | Carbon / AppKit 事件监听 | 首版优先选择实现成本低且稳定的方案 |
| 鼠标位置 | NSEvent / CGEvent | 获取鼠标位置和方向 |
| 动作执行 | NSWorkspace / Process | 打开 URL、应用和命令 |
| 配置管理 | Codable + JSON | 读写本地配置 |
| 测试 | XCTest | 角度计算、配置解析、动作参数校验 |

### 10.2 模块划分

| 模块 | 职责 |
|---|---|
| QuickNavApp | 应用入口、生命周期管理 |
| AppDelegate | 菜单栏、Dock 策略、系统级初始化 |
| StatusBarController | 菜单栏状态和菜单操作 |
| HotKeyManager | 注册和监听全局快捷键 |
| MouseTracker | 获取鼠标位置和移动事件 |
| RadialWindowController | 创建、显示、隐藏浮动窗口 |
| RadialMenuView | 圆形菜单主 UI |
| RadialMenuItemView | 单个菜单项 UI |
| MenuState | 当前菜单中心点、选中项、可见状态 |
| AngleCalculator | 角度计算和菜单项命中 |
| ActionExecutor | 根据动作类型执行操作 |
| ConfigManager | 默认配置、用户配置、配置重载 |
| PermissionManager | 权限检查和跳转引导 |
| Logger | 日志输出 |

## 11. 版本规划

### 11.1 V0.1 原型

目标：验证圆形菜单交互是否可用。

范围：

- 创建 Swift macOS 项目。
- 显示圆形菜单窗口。
- 使用模拟数据渲染 4 到 8 个菜单项。
- 根据鼠标位置计算方向。
- 当前选中项高亮。
- 松开触发时打印动作 ID。
- 增加 `AngleCalculator` 单元测试。

不包含：

- 全局快捷键。
- 菜单栏常驻。
- 真实打开 URL 或应用。
- JSON 配置。

### 11.2 V0.2 可用版本

目标：形成可日常使用的最小版本。

范围：

- 全局快捷键唤起和释放。
- 菜单栏常驻。
- 打开 URL。
- 启动本地应用。
- 读取 JSON 配置。
- 配置重载。
- 基础错误日志。

### 11.3 V0.3 优化版本

目标：提升稳定性和可配置性。

范围：

- 设置窗口。
- 快捷键配置。
- 权限状态提示。
- 动画优化。
- 命令动作支持。
- 动作失败提示。
- 更完整的配置校验。

### 11.4 V1.0 稳定版本

目标：稳定常驻，满足长期个人工作流。

范围：

- 完整配置化管理。
- 支持多动作类型。
- 支持基础 UI 自定义。
- 稳定错误处理。
- 多显示器适配。
- 打包发布本地安装包。

## 12. 验收标准

### 12.1 V0.1 验收标准

- 能在 macOS 上运行并显示圆形菜单。
- 菜单项围绕中心点均匀分布。
- 鼠标移动到不同方向时，高亮项正确变化。
- 鼠标回到中心取消区域时，无选中项。
- 触发动作时能打印正确动作 ID。
- `AngleCalculator` 对 4、6、8 项菜单的命中结果有测试覆盖。

### 12.2 V0.2 验收标准

- 应用启动后出现在菜单栏。
- 按下默认快捷键能在鼠标位置显示菜单。
- 松开快捷键后菜单关闭。
- 选中 URL 动作后能打开浏览器。
- 选中 app 动作后能启动对应应用。
- 配置文件修改后，通过菜单栏重载能生效。
- 配置损坏时应用不崩溃，并使用默认配置。

### 12.3 V1.0 验收标准

- 应用可连续常驻运行，不出现菜单残留或快捷键失效。
- 常用动作执行稳定，失败可定位原因。
- 多显示器下菜单显示在当前鼠标所在屏幕。
- 用户可以通过配置或设置页完成日常菜单维护。

## 13. 风险和待确认问题

### 13.1 技术风险

- macOS 全局快捷键和事件监听受权限影响，需要处理授权引导。
- Control + Space 可能与系统输入法或 Spotlight 配置冲突，需要支持替换快捷键。
- 不同 macOS 版本对透明置顶窗口和事件监听行为可能存在差异。
- `command` 动作具备执行本地命令能力，需明确仅面向个人本地使用。

### 13.2 待确认问题

- 默认菜单项数量使用 6 个还是 8 个。
- 角度排列方向采用顺时针还是逆时针。
- 默认快捷键是否保留 Control + Space。
- 首版是否需要隐藏 Dock 图标。
- V0.2 是否纳入 `command` 动作，还是推迟到 V0.3。
